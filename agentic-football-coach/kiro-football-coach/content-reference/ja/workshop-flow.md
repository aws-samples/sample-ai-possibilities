# ワークショップフロー — エージェンティックフットボールワールドカップ

このドキュメントでは、初期セットアップから競技プレーまでの9フェーズのワークショップ進行について説明します。

## フェーズ1：ウェルカム＆セットアップ

参加者はAWS Workshop Studioアクセス（12桁のイベントアクセスコード）を受け取り、OTPで認証し、AWS CLI認証情報を設定します。ワークショップはus-east-1またはus-west-2で実行されます。

**Kiro IDEセットアップ：**
1. [kiro.dev](https://kiro.dev)からKiroをインストール
2. AWSコンソールでIAM Identity Centerユーザーを設定（ユーザー名、メール、名前を入力）
3. Identity Centerのメール招待を承認し、パスワードを作成
4. Identity CenterのMFAを無効化（設定→認証→多要素認証→なし）
5. AWSコンソールでKiroを検索し、設定からKiro Proサブスクリプションでユーザーを追加
6. Kiro IDEでIAM Identity Centerのサインイン URLを使用してサインイン

**Kiro Football Coachセットアップ：**
参加者はKiroチャットにセットアッププロンプトを貼り付けて、コーチパッケージ（ステアリングファイル、スキル、フック、エージェント、コンテンツリファレンス）をダウンロードして設定できます。

**主な成果:** AWSコンソールアクセス、CLI認証情報の設定完了、Kiro IDEのインストールとサインイン完了、コーチ設定完了、環境の準備完了。

## フェーズ2：AIエージェントの基礎

AIエージェントの紹介 — LLMを「頭脳」として使用し、ツールやアクションについて判断を下す自律型ソフトウェアエンティティ。エージェントのスペクトラムをカバー：

- **特化型エージェント** — 迅速なデプロイのための構築済みソリューション（例：Amazon Q Developer）
- **エンタープライズグレードエージェント** — オープンソースフレームワーク + AgentCoreマネージドインフラストラクチャ
- **フルマネージドエージェント** — 最小限のコードでAmazon Bedrock Agents
- **セルフマネージドエージェント** — StrandsやLangGraphなどのフレームワークによる完全DIY

コアエージェントシステムコンポーネント：基盤モデル、オーケストレーション、ツール統合、メモリとコンテキスト、ガードレールと安全性、モニタリングとオブザーバビリティ。

**主な成果:** エージェントタイプの理解、コアエージェントシステムアーキテクチャ、システムプロンプト、状態管理。

## フェーズ3：フットボールの基礎

5v5 AI形式に適応したフットボールの基礎の紹介：

- **ポジション:** ゴールキーパー、ディフェンダー、ミッドフィルダー、フォワード
- **フィールドゾーン:** ディフェンシブサード、ミッドフィールド、アタッキングサード、ペナルティボックス、ゴールエリア
- **空間概念:** 幅、深さ、スペーシング、ポジショニング
- **基本ルール:** ハンド禁止（ゴールキーパー除く）、ファウル→フリーキック、オフサイドルール

参加者は実際のフットボール概念が5v5シミュレーションでのエージェント意思決定にどのようにマッピングされるかを学びます。

**主な成果:** フットボールのポジション、フィールドレイアウト、基本ルールの理解。

## フェーズ4：フレームワーク選択

参加者は2つのエージェントフレームワークから選択します：

| 考慮事項 | Strands Agents SDK | LangChain SDK |
|---|---|---|
| 抽象化レベル | 低い — エージェントループの明示的制御 | 高い — 確立されたエコシステム |
| 最適な用途 | カスタムワークフロー、きめ細かいロジック | 構造化されたマルチステップ推論 |
| 経験レベル | 手動オーケストレーションに慣れている | LangChainの規約に精通 |

両方ともAmazon Bedrock AgentCoreにデプロイ可能。コーチは両方を中立的に提示し、参加者の経験と目標に基づいて選択を支援します。

**主な成果:** フレームワークの選択完了、トレードオフの理解。

## フェーズ5：エージェントの構築とデプロイ

参加者はGitHubからサンプルエージェントリポジトリをクローンしてAgentCoreにデプロイします：

1. **リポジトリのクローン** — Strands: `git clone https://github.com/aws-samples/fifa-agentic-football-strands.git`。LangChain: `git clone https://github.com/aws-samples/fifa-agentic-football-langchain.git`
2. **環境のセットアップ** — Python venvの作成、`pip install -r requirements.txt`で依存関係のインストール
3. **ローカルテスト** — `agentcore dev`でローカル開発サーバーを起動、`agentcore invoke --dev`でテスト呼び出し
4. **デプロイ** — `./deploy-local.sh`を実行（IAMロール作成、依存関係パッケージング、S3アップロード、エージェントデプロイを処理）
5. **Agent ARNを保存** — デプロイ出力のARN形式：`arn:aws:bedrock-agentcore:<region>:<account-id>:runtime/<agent-id>`

サンプルエージェントのデモ内容：BedrockAgentCoreAppランタイム、システムプロンプト付きStrands/LangChainエージェント、ゲーム状態サマライザー、コマンドパーサー、ルールベースフォールバック。

**主な成果:** AgentCoreにデプロイされたサンプルエージェント、Agent ARN取得。

## フェーズ6：登録とプレー

参加者はプレイヤーポータルでエージェントを登録し、マッチをプレーします：

1. ワークショップインストラクターから提供された認証情報でプレイヤーポータルにログイン
2. チームを作成または参加
3. トーナメントに登録
4. エージェントを登録（エージェント名とAgentCore ARNを入力、5人のプレイヤー全員に繰り返し）
5. トレーニングマッチをプレー（Kiro Phantomsベンチマークチーム対戦）
6. ライブ2Dビューアーで観戦
7. ライブマッチ中にコーチ指示を送信

**主な成果:** エージェント登録完了、トレーニングマッチプレー完了、コーチングシステムの理解。

## フェーズ6：エージェントの強化

参加者はフレームワーク固有のパターンを使用してエージェントを改善します：

**Strands SDKの強化：**
- 構造化出力（バリデーション済みアクション用のPydanticモデル）
- カスタムツール（戦術分析関数）
- オブザーバビリティ（メトリクス、トレース、レイテンシ追跡）
- フック（モニタリング、ツール呼び出し制限）
- モデル切り替え（Nova Micro vs Claude）
- ガードレール（コンテンツフィルタリング）
- 状態追跡（相手パターン）
- リトライ戦略（スロットリング処理）
- マルチエージェントパイプライン（分析→計画→実行）

**LangGraphの強化：**
- ガードレール統合
- プロンプト改善
- テスト強化
- パフォーマンスチューニング
- 追加ツール
- マルチエージェントアーキテクチャ

**主な成果:** 意思決定、オブザーバビリティ、レジリエンスが改善されたエージェント。

## フェーズ8：マッチデー

参加者はライブマッチビューアーとコーチングシステムを使用します：

- ライブ2Dビューアー（青ドット=チームA、赤ドット=チームB、白ドット=ボール、金リング=ボール保持者）
- コーチ指示パネルでリアルタイム戦術指示
- 指示は次のティックで有効 — 再デプロイ不要

コード変更後は`./deploy-local.sh`で再デプロイ（ARNは同じまま）。

**主な成果:** ライブマッチ観戦、リアルタイムコーチング実践。

## フェーズ9：競技

参加者はトーナメントブラケットに参加します：

- マッチは5v5、5分間、60ティック/秒
- スコアリング：最多ゴールが勝利
- リーダーボード：勝利=3ポイント、引き分け=1ポイント、敗北=0ポイント
- LLMの非決定性を活用するため複数回のトレーニングマッチが可能（結果はラン間で異なる）
- フットボール戦略の適用：ポゼッションプレー、カウンターアタック、ハイプレス、ウイングプレー
- 守備戦略の適用：ゾーンディフェンス、マンツーマンマーキング、コンパクトディフェンス、オフサイドトラップ
- マッチ全体を通じたスタミナ管理

**主な成果:** 競技マッチのプレー完了、反復を通じた戦略の洗練。
