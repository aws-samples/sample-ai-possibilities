# 워크숍 흐름 — 에이전틱 풋볼 월드컵

이 문서는 초기 설정부터 경쟁 플레이까지의 9단계 워크숍 진행 과정을 설명합니다.

## 1단계: 환영 및 설정

참가자는 AWS Workshop Studio 액세스(12자리 이벤트 액세스 코드)를 받고, OTP로 인증하며, AWS CLI 자격 증명을 설정합니다. 워크숍은 us-east-1 또는 us-west-2에서 실행됩니다.

**Kiro IDE 설정:**
1. [kiro.dev](https://kiro.dev)에서 Kiro 설치
2. AWS 콘솔에서 IAM Identity Center 사용자 설정 (사용자 이름, 이메일, 이름 입력)
3. Identity Center 이메일 초대 수락 및 비밀번호 생성
4. Identity Center MFA 비활성화 (설정 → 인증 → 다단계 인증 → 안 함)
5. AWS 콘솔에서 Kiro 검색, 설정에서 Kiro Pro 구독으로 사용자 추가
6. Kiro IDE에서 IAM Identity Center 로그인 URL을 사용하여 로그인

**Kiro Football Coach 설정:**
참가자는 Kiro 채팅에 설정 프롬프트를 붙여넣어 코치 패키지(스티어링 파일, 스킬, 훅, 에이전트, 콘텐츠 참조)를 다운로드하고 설정할 수 있습니다.

**주요 성과:** AWS 콘솔 액세스, CLI 자격 증명 설정 완료, Kiro IDE 설치 및 로그인 완료, 코치 설정 완료, 환경 준비 완료.

## 2단계: AI 에이전트 기초

AI 에이전트 소개 — LLM을 "두뇌"로 사용하여 도구와 액션에 대한 결정을 내리는 자율 소프트웨어 엔티티. 에이전트 스펙트럼 소개:

- **특화 에이전트** — 빠른 배포를 위한 사전 구축 솔루션 (예: Amazon Q Developer)
- **엔터프라이즈급 에이전트** — 오픈소스 프레임워크 + AgentCore 관리형 인프라
- **완전 관리형 에이전트** — 최소한의 코드로 Amazon Bedrock Agents
- **자체 관리형 에이전트** — Strands나 LangGraph 같은 프레임워크를 사용한 완전 DIY

핵심 에이전트 시스템 구성 요소: 기반 모델, 오케스트레이션, 도구 통합, 메모리 및 컨텍스트, 가드레일 및 안전, 모니터링 및 관측성.

**주요 성과:** 에이전트 유형 이해, 핵심 에이전트 시스템 아키텍처, 시스템 프롬프트, 상태 관리.

## 3단계: 풋볼 기초

5v5 AI 형식에 맞춘 풋볼 기초 소개:

- **포지션:** 골키퍼, 수비수, 미드필더, 공격수
- **필드 구역:** 수비 지역, 미드필드, 공격 지역, 페널티 박스, 골 에리어
- **공간 개념:** 폭, 깊이, 간격, 포지셔닝
- **기본 규칙:** 핸드 금지(골키퍼 제외), 파울 → 프리킥, 오프사이드 규칙

참가자는 실제 풋볼 개념이 5v5 시뮬레이션에서 에이전트 의사 결정에 어떻게 매핑되는지 배웁니다.

**주요 성과:** 풋볼 포지션, 필드 레이아웃, 기본 규칙 이해.

## 4단계: 프레임워크 선택

참가자는 두 가지 에이전트 프레임워크 중 하나를 선택합니다:

| 고려 사항 | Strands Agents SDK | LangChain SDK |
|---|---|---|
| 추상화 수준 | 낮음 — 에이전트 루프의 명시적 제어 | 높음 — 확립된 에코시스템 |
| 최적 용도 | 커스텀 워크플로, 세밀한 로직 | 구조화된 다단계 추론 |
| 경험 수준 | 수동 오케스트레이션에 익숙함 | LangChain 규칙에 익숙함 |

두 프레임워크 모두 Amazon Bedrock AgentCore에 배포 가능합니다. 코치는 두 가지를 중립적으로 제시하고 참가자의 경험과 목표에 따라 선택을 도와줍니다.

**주요 성과:** 프레임워크 선택 완료, 트레이드오프 이해.

## 5단계: 에이전트 구축 및 배포

참가자는 GitHub에서 샘플 에이전트 리포지토리를 클론하고 AgentCore에 배포합니다:

1. **리포지토리 클론** — Strands: `git clone https://github.com/aws-samples/fifa-agentic-football-strands.git`. LangChain: `git clone https://github.com/aws-samples/fifa-agentic-football-langchain.git`
2. **환경 설정** — Python venv 생성, `pip install -r requirements.txt`로 의존성 설치
3. **로컬 테스트** — `agentcore dev`로 로컬 개발 서버 시작, `agentcore invoke --dev`로 테스트 호출
4. **배포** — `./deploy-local.sh` 실행 (IAM 역할 생성, 의존성 패키징, S3 업로드, 에이전트 배포 처리)
5. **Agent ARN 저장** — 배포 출력의 ARN 형식: `arn:aws:bedrock-agentcore:<region>:<account-id>:runtime/<agent-id>`

샘플 에이전트 데모 내용: BedrockAgentCoreApp 런타임, 시스템 프롬프트가 있는 Strands/LangChain 에이전트, 게임 상태 요약기, 명령 파서, 규칙 기반 폴백.

**주요 성과:** AgentCore에 배포된 샘플 에이전트, Agent ARN 획득.

## 6단계: 등록 및 플레이

참가자는 플레이어 포털에서 에이전트를 등록하고 매치를 플레이합니다:

1. 워크숍 강사가 제공한 자격 증명으로 플레이어 포털에 로그인
2. 팀 생성 또는 참가
3. 토너먼트 등록
4. 에이전트 등록 (에이전트 이름과 AgentCore ARN 입력, 5명의 플레이어 모두에 대해 반복)
5. 트레이닝 매치 플레이 (Kiro Phantoms 벤치마크 팀 대전)
6. 라이브 2D 뷰어로 관전
7. 라이브 매치 중 코치 지시 전송

**주요 성과:** 에이전트 등록 완료, 트레이닝 매치 플레이 완료, 코칭 시스템 이해.

## 6단계: 에이전트 강화

참가자는 프레임워크별 패턴을 사용하여 에이전트를 개선합니다:

**Strands SDK 강화:**
- 구조화된 출력 (검증된 액션을 위한 Pydantic 모델)
- 커스텀 도구 (전술 분석 함수)
- 관측성 (메트릭, 트레이스, 레이턴시 추적)
- 훅 (모니터링, 도구 호출 제한)
- 모델 전환 (Nova Micro vs Claude)
- 가드레일 (콘텐츠 필터링)
- 상태 추적 (상대 패턴)
- 재시도 전략 (스로틀링 처리)
- 멀티 에이전트 파이프라인 (분석 → 계획 → 실행)

**LangGraph 강화:**
- 가드레일 통합
- 프롬프트 개선
- 테스트 강화
- 성능 튜닝
- 추가 도구
- 멀티 에이전트 아키텍처

**주요 성과:** 의사 결정, 관측성, 복원력이 개선된 에이전트.

## 8단계: 매치 데이

참가자는 라이브 매치 뷰어와 코칭 시스템을 사용합니다:

- 라이브 2D 뷰어 (파란 점=팀A, 빨간 점=팀B, 흰 점=볼, 금색 링=볼 소유자)
- 코치 지시 패널로 실시간 전술 지시
- 지시는 다음 틱에서 적용 — 재배포 불필요

코드 변경 후 `./deploy-local.sh`로 재배포 (ARN은 동일하게 유지).

**주요 성과:** 라이브 매치 관전, 실시간 코칭 실습.

## 9단계: 경쟁

참가자는 토너먼트 브래킷에 참가합니다:

- 매치는 5v5, 5분, 60틱/초
- 득점: 최다 골이 승리
- 리더보드: 승리=3포인트, 무승부=1포인트, 패배=0포인트
- LLM의 비결정성을 활용하기 위해 여러 번 트레이닝 매치 가능 (결과는 실행마다 다름)
- 풋볼 전략 적용: 점유율 플레이, 역습, 하이 프레스, 윙 플레이
- 수비 전략 적용: 지역 수비, 대인 마크, 컴팩트 수비, 오프사이드 트랩
- 매치 전체에 걸친 스태미나 관리

**주요 성과:** 경쟁 매치 플레이 완료, 반복을 통한 전략 개선.
